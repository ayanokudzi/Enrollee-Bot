# Design-Doc: Чат бот "Enrollment 2026" / RAG-система
**Проектная команда:**
* **Бизнес-заказчик:** Факультет компьютерных наук НИУ ВШЭ (Приемная комиссия).
* **Разработчик чат-бота:** Ковязин Алексей Дмитриевич.
## 1. Контекст проекта
### 1.1 Бизнес-задача
* у абитуриентов много вопросов и мало ответов, разобраться в многочисленных справочниках и документах будет далеко не просто. приемная комиссия может быть перегружена в связи с большим потоком поступающих.
* создать обычного бота с кнопками не получится - количество вопросов от абитуриентов неисчислимо велико, поэтому используем rag систему. также выбор системы обусловлен динамичностью данных (достаточно обновить текстовый файл в базе знаний) и точностью ответов (система идет строго по официальным документам).
* успех проявляется в облегчении работы приемной коммиссии, сокращении времени ответа.
### 1.2 Целевая аудитория и пользователи
* Чат-ботом будут пользоваться все кому интересно поступление - от абитуриентов до их родителей.
* Взаимодействие с ботом просиходит в мессенджере Telegram. Выбор обучен тем, что это основной канал коммуникации абитуриентов и студентов ФКН. Абитуриент задает вопрос в свободной форме, а бот трансформирует сложный язык официальных документов в простой и понятный ответ.
* Ожидаемая нагрузка 5000-10000 пользователей за сезон. Пиковая нагрузка с 12:00 по 18:00 (UTC+3).
### 1.3 Ограничения и допущения
* Бот не будет поддерживать пустое общение, будут обсуждаться только вопросы с поступлением. Также бот не будет давать прогнозов по проходным баллам, только статистика прошлых лет.
* Ответы бота не являются официальной офертой.
## 2. Архитектура решения
### 2.1 Общая схема
* Telegram Interface (Принимает сообщение) -> Backend (Python) (чистит запрос и отправляет в RAG движок) -> Retrieval (Стучимся в векторную БД) -> LLM Prompt (В коде лежит шаблон промпта. В него вставляется вопрос + найденный текст.) -> Generation (LLM дает ответ) -> Post-processing (Бот добавляет ссылку на источник и отправляет пользователю)
### 2.2 Хранилище знаний / документооборот
* Источниками являются PDF/тексты с официального сайта ФКН.
* Векторное представление хранится в базе FAISS. Исходные тексты лежат в папке data/.
* Обновление ручное.
### 2.3 Retrieval + Generation
* Retrieval: Модель sentence-transformers
* Generation: Использование API (например, GPT-3.5/4)
### 2.4 Интеграции и интерфейсы
* Интерфейс: Telegram Bot API
* Все диалоги сохраняются в текстовый лог
* Кнопки быстрого доступа к популярным разделам
### 2.5 Инфраструктура и развертывание
* Развертывание происходит через виртуальный сервер (хостер vdsina). Имеется готовый сервер с CPU 1 core, 2GB RAM.
